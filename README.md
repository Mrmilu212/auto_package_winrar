# auto_package_winrar

一个 Windows 图形界面工具：通过拖放文件/文件夹，自动调用 **WinRAR** 进行压缩或解压，支持多次压缩、伪装后缀、分卷压缩以及智能递归解压。

---

## 功能概览

### 压缩功能

- **拖放打包**：拖放文件或文件夹到窗口即可开始压缩
- **一次压缩**：压缩包内容与拖放对象一致
  - 拖放文件夹：压缩包里包含该**文件夹本身**
  - 拖放文件：压缩包里只包含该**文件本身**
- **二次压缩**：将第一次压缩产物改为随机"伪装后缀"（如 `.png`、`.dll`、`.mp4` 等）后，再对该伪装文件进行打包
- **三次压缩（分卷）**：在二次压缩基础上继续打包，并按规则分卷
  - **至少 2 卷**
  - **每卷最大 2049MB**
  - 单卷大小 ≈ 总大小/2 + 10MB
- **随机命名压缩包**：每进行一次压缩都会生成 **5 位**随机文件名（`A-Za-z0-9`）
  - 所有压缩模式的最终产物都会放入**随机命名的文件夹**中
  - 三次分卷最终输出形如：`ABCDE` 文件夹包含 `ABCDE.part1.rar`、`ABCDE.part2.rar` ...

### 解压功能

- **智能识别格式**：自动识别 `.rar`、`.7z`、`.zip` 三种压缩格式
- **去伪装解压**：自动识别伪装后缀文件，尝试解压
- **递归解压**：智能判断是否需要继续解压嵌套的压缩包
- **最终文件判断规则**：
  - 包含 `.exe` 可执行文件 → 最终文件
  - 有 2 个及以上非压缩格式文件 → 最终文件
  - 只有 1 个非压缩格式文件 → 仅尝试 `.zip` 解压，失败则最终文件
  - 同时包含文件和文件夹 → 最终文件
- **原子性操作**：解压过程失败时自动回滚，不留下半成品
- **结果整理**：自动找到最终内容所在的最深层目录，使用该目录的名称作为目标文件夹名称

### 界面功能

- **双拖放区域**：左侧压缩区，右侧解压区
- **窗口大小记忆**：自动保存和恢复窗口大小
- **已用时间显示**：状态栏实时显示已用时间和当前阶段
- **取消操作**：压缩/解压过程中可随时取消
- **日志显示**：实时显示操作日志

## 环境要求

- **操作系统**：Windows 10/11
- **WinRAR**：已安装 WinRAR，且存在命令行工具
  - `Rar.exe`（用于压缩）
  - `WinRAR.exe`（用于解压，支持 rar/7z/zip）
  - 常见路径：`C:\Program Files\WinRAR\`
- **Python**：Python 3.10+，并确保能使用 `pythonw.exe`（用于无控制台启动）

## 使用说明

### 压缩操作

1. 打开程序窗口
2. 默认已开启"三次压缩（分卷）"模式
3. 将文件/文件夹拖放到**左侧压缩拖放区**
4. 等待压缩完成

**输出位置**：默认输出到拖放对象所在目录（生成随机 5 位名称的文件夹，包含压缩产物）

### 解压操作

1. 将要解压的文件拖放到**右侧解压拖放区**
2. 程序自动识别格式并解压
3. 解压完成后，文件放在以最终内容目录名称命名的文件夹中

**支持的格式**：
- `.rar` - RAR 压缩包
- `.7z` - 7-Zip 压缩包
- `.zip` - ZIP 压缩包
- 伪装后缀文件（自动尝试解压）

## 常见问题

### 未找到 WinRAR

- 请确认已安装 WinRAR
- 或将 WinRAR 安装目录加入系统 PATH

### 双击启动提示找不到 pythonw

- 请安装 Python，并在安装时勾选 **Add Python to PATH**
- 或手动把 Python 安装目录加入 PATH（其中应包含 `pythonw.exe`）

### 解压失败

- 确认文件是否为有效的压缩包
- 确认 WinRAR.exe 是否存在
- 查看日志区域的错误信息

## 技术说明

### 压缩流程

```
用户拖放 → 生成随机文件名 → 调用 Rar.exe 压缩 → 
[二次] 改名伪装后缀 → 再次压缩 → 
[三次] 再次伪装 → 分卷压缩 → 输出
```

### 解压流程

```
用户拖放 → 识别格式 → 调用 WinRAR.exe 解压 → 
递归检查内容 → 判断是否继续解压 → 
整理文件到文件夹 → 输出
```

### 原子性保证

- 所有操作在临时目录进行
- 只有全部成功才移动到目标目录
- 失败时自动清理临时文件
- 使用 `finally` 块确保资源释放
